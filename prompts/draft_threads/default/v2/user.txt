아래 입력 JSON(payload_json)만을 근거로 Threads 5단 스레드 초안을 작성하라. 다른 가정/외부지식/추가 질문 없이 작성한다.

[입력]
- payload_json에는 intake, selected_candidate, normalized_brief(있을 수 있음)가 포함된다.
- intake 필드: industry, target_role, issue_stage, pain_picker(1~2), content_goal, offer_material, pain_sentence?, experience_seed?, must_avoid?
- selected_candidate: 주제/각도/핵심 메시지(필드명이 무엇이든 의미를 파악해 활용)
- must_avoid는 추가 금지어 목록으로 취급한다.

[스타일(10개 유형) 선택]
- 우선순위:
  1) selected_candidate에 thread_type/style/type 등이 있으면 그것을 따른다.
  2) 없으면 content_goal과 issue_stage로 추론:
     - issue_stage가 “긴급/이슈발생/뉴스/판결/개정” 성격이면: 3) 시의성 브리핑
     - content_goal이 “리드/자료요청/전환”이고 offer_material이 있으면: 10) 리드 전환 + 5) 체크리스트 혼합
     - pain_picker가 “실수 방지/체크/준비” 성격이면: 5) 체크리스트/팁
     - target_role이 “CEO/CFO/사내변/CTO”이고 의사결정 관점이면: 4) 관점/에세이
     - 질문형 주제면: 2) 법률 Q&A
     - 그 외 기본값: 4) 관점/에세이(파트너 보이스)
- 어떤 유형을 선택해도 [1/5]~[5/5] 구조는 유지한다.

[작성 규칙]
- 언어: 한국어, 비즈니스 톤. 문장 간결(Threads).
- “Hook–Insight–Context–Soft CTA”가 자연스럽게 흐르도록 5줄에 분배한다.
- pain_picker와 pain_sentence가 있으면 [1/5] 또는 [3/5]에서 ‘리스크/비용/시간’ 언어로 재진술한다.
- experience_seed가 있으면 “익명화된 실무 패턴”으로만 사용(식별 금지).
- 법적 근거는 [2/5]에서 1~2문장으로 짧게(조문번호 장문 금지).
- [4/5] 체크리스트는 한 줄 안에서 3~5개 항목(①②③④⑤ 또는 “/”로 구분).
- 외부 URL 직접 기재 금지. CTA는 “프로필/DM/양식 안내”로만.
- CTA는 1개만. (예: “댓글에 ‘자료’라고 남기면 DM” 또는 “프로필 안내 참고” 중 하나)
- [5/5]에는 짧은 디스클레이머를 자연스럽게 포함:
  예) “일반 정보이며, 개별 사안은 사실관계에 따라 달라질 수 있습니다.”

[출력 스키마(엄수)]
- JSON ONLY로 출력. 키는 정확히 title_candidates, body_md_lines 두 개만.
- title_candidates: 3~6개. 과장/최상급/성과보장/무료 유인/전문 표기 금지.
- body_md_lines: 정확히 5개 요소.
- 각 요소는 반드시 한 줄이며 개행문자 포함 금지.
- 각 줄은 순서대로 [1/5]...[5/5]로 시작해야 한다.

[최종 점검(출력 직전 self-check)]
- JSON 문법 정상? 두 키만 존재?
- body_md_lines 정확히 5줄? [i/5] 접두어 모두 존재?
- 금지어 및 must_avoid 단어 포함 없음?
- 직접 URL 없음? CTA 1개만 존재?
- 식별 가능한 디테일(지역/날짜/금액/회사명) 없음?
- [5/5]에 디스클레이머 포함?
- 위반 시 스스로 수정 후 출력.

입력(JSON): {{payload_json}}
