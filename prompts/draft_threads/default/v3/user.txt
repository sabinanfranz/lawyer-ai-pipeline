아래 입력 JSON(payload_json)만을 근거로 Threads 3포스트 초안을 작성하라. 다른 가정/외부지식/추가 질문 없이 작성한다.

[입력]
- payload_json에는 intake, selected_candidate, normalized_brief(있을 수 있음)가 포함된다.
- intake 필드: industry, target_role, issue_stage, pain_picker(1~2), content_goal, offer_material, pain_sentence?, experience_seed?, must_avoid?
- selected_candidate: 주제/각도/핵심 메시지(필드명이 무엇이든 의미를 파악해 활용)
- must_avoid는 추가 금지어 목록으로 취급한다.

[스타일(10개 유형) 선택]
- 우선순위:
  1) selected_candidate에 thread_type/style/type 등이 있으면 그것을 따른다.
  2) 없으면 content_goal과 issue_stage로 추론:
     - issue_stage가 “긴급/이슈발생/뉴스/판결/개정” 성격이면: 3) 시의성 브리핑
     - pain_picker가 “실수 방지/체크/리스크” 성격이면: 5) 리스크 알림·체크리스트
     - pain_picker가 “절차/프로세스/순서/증거 준비” 성격이면: 6) 프로세스·절차 안내
     - content_goal이 “리드/자료요청/전환”이고 offer_material이 있으면: 10) 리드 전환(Soft CTA)
     - target_role이 “CEO/CFO/사내변/CTO”이고 의사결정 관점이면: 4) 관점/에세이
     - 질문형 주제면: 2) 법률 Q&A
     - 그 외 기본값: 4) 관점/에세이(파트너 보이스)

[톤/말투(강제)]
- 친절하게 소개하는 말투로 시작한다(예: “안녕하세요, 실무에서 자주 나오는 포인트를 짧게 정리해볼게요.”).
- 각 포스트에 최소 1개 이상 “상황 예시”를 섞는다(단, 식별정보 금지). 예: “한 HR팀이 퇴사자 계정 정리를 미뤘다가…”, “한 스타트업이 외주 검수 문구를 놓쳐서…”.
- 독자를 ‘실무자/의사결정자’로 가정하고, “비용/시간/리스크/협상력” 언어로 번역한다.

[본문(3개) 구성 규칙]
- body_md_lines는 반드시 정확히 3개 요소.
- 각 요소는 순서대로 [1/3], [2/3], [3/3] 로 시작.
- 각 요소(포스트) 내부는 문단 2개 이상 포함(문단 구분은 반드시 "\n\n" 사용. 실제 줄바꿈 문자 직접 입력 금지).
- 각 문단은 3~5문장으로 구성.
- 문장은 간결하게(Threads 특성). 긴 문장/장문 인용 금지.
- 직접 URL 금지. “프로필/DM/양식 안내”로만 표현.
- CTA는 전체 글에서 1개만(권장: [3/3]에만). “무료/할인/유인” 표현 금지.
- 댓글 대화를 위해 전체 글에서 질문 최소 1개 포함(권장: [3/3] 마지막 문단에 배치).

[Hook–Insight–Context–Soft CTA 배치(권장)]
- [1/3] Hook + 문제 제기 + 상황 예시(익명) + 왜 중요한지(리스크/비용/시간)
- [2/3] 원칙/기준(짧게) + 실무 리스크 패턴(익명 예시) + 흔한 오해 1개(있으면)
- [3/3] 대응 포인트 3~5개를 “문장 속 체크포인트”로 정리 + Soft CTA(자료/체크리스트/가이드) + 질문 + 디스클레이머(마지막 문장)

[디스클레이머(필수)]
- [3/3]의 마지막 문장에 반드시 포함:
  “일반 정보 제공 목적이며, 개별 사안은 사실관계에 따라 달라질 수 있습니다.”

[출력 스키마(엄수)]
- JSON ONLY로 출력. 키는 정확히 title_candidates, body_md_lines 두 개만.
- title_candidates: 3~6개. 과장/최상급/성과보장/무료 유인/전문 표기 금지.
- body_md_lines: 정확히 3개 요소.
- 각 요소는 JSON 문자열 1개이며, 문단 구분은 "\n\n" 이스케이프만 사용(실제 줄바꿈 문자 금지).

[최종 점검(출력 직전 self-check)]
- JSON 문법 정상? 두 키만 존재?
- body_md_lines 정확히 3개? [i/3] 접두어 모두 존재?
- 각 포스트에 문단 2개 이상? 각 문단 3~5문장?
- 금지어 및 must_avoid 포함 없음?
- 직접 URL 없음? CTA 1개만 존재?
- 식별 가능한 디테일(지역/날짜/금액/회사명) 없음?
- [3/3] 마지막 문장에 디스클레이머 포함?
- 위반 시 스스로 수정 후 출력.

입력(JSON): {{payload_json}}
