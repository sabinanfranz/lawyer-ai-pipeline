이전 출력이 스키마/규칙을 위반했다. 아래 요구를 100% 충족하는 “유효 JSON ONLY”로 즉시 재작성하라.

[반드시 지킬 것]
- 출력은 JSON 오브젝트 1개만. 코드펜스/설명/텍스트 금지.
- 키는 정확히 2개만: "title_candidates", "body_md_lines"
- title_candidates: 3~6개 문자열 배열(비어있으면 안 됨)
- body_md_lines: 정확히 3개 문자열 배열(비어있으면 안 됨)
- 각 body_md_lines 요소는 반드시 [1/3], [2/3], [3/3] 순서로 시작
- 각 요소 내부에 문단 2개 이상 포함. 문단 구분은 반드시 "\n\n" 이스케이프 사용(실제 줄바꿈 문자 금지).
- 각 문단은 3~5문장.
- 직접 URL 금지. “프로필/DM/양식 안내”만 허용.
- CTA는 전체에서 1개만(권장: [3/3]). 무료/할인/유인/성과보장 표현 금지.
- 금지어(전문/최상급/결과보장/무료/할인/전관/비방/비교/식별정보) 및 must_avoid(주어졌다면) 모두 제거.
- [3/3] 마지막 문장에 반드시 디스클레이머 포함:
  “일반 정보 제공 목적이며, 개별 사안은 사실관계에 따라 달라질 수 있습니다.”

[정보가 부족할 때]
- 주제가 불명확해도 “기업 실무에서 흔한 리스크”를 일반화해 작성하되, 확정/보장 표현 없이 보수적으로 쓴다.
- 그래도 반드시 3개 제목 + 3개 본문(각 2문단 이상)을 채워 유효 JSON을 만든다.
